1. Funcionalidades Principales

1.1 Conversión de Monedas

El sistema permite la conversión entre dólares (USD) y soles (PEN) con tasas de cambio en tiempo real:

  • Conversión de USD a PEN (compra/venta)
  • Conversión de PEN a USD (compra/venta)
  • Actualización en tiempo real de las tasas
  • Interfaz intuitiva con pestañas para compra/venta

1.2 Manejo de Estados

El estado de la aplicación se gestiona mediante Pinia, con un store dedicado para las tasas de cambio:

// store/exchange.ts
import { defineStore } from 'pinia';
import { ref, computed } from 'vue';

export const useExchangeStore = defineStore('exchange', () => {
  const purchasePrice = ref(0);  // Tasa de compra
  const salePrice = ref(0);      // Tasa de venta
  const loading = ref(true);
  const error = ref(null);

  // Getters computados
  const rates = computed(() => ({
    purchase: purchasePrice.value,
    sale: salePrice.value
  }));

  // Funciones de conversión
  function convertToSoles(amount: number): number {
    return amount * purchasePrice.value;
  }

  function convertToDollars(amount: number): number {
    return amount / salePrice.value;
  }

  return {
    purchasePrice,
    salePrice,
    loading,
    error,
    rates,
    convertToSoles,
    convertToDollars
  };
});

2. Implementación Técnica

2.1 Reactividad en el Conversor

La aplicación utiliza el sistema de reactividad de Vue 3 de manera eficiente:

// En CurrencyConverter.vue
const exchangeStore = useExchangeStore();
const activeTab = ref('Compra');
const amount = ref('');
const result = ref('');

// Actualizar resultado cuando cambia el monto o la pestaña
watch([amount, activeTab], () => {
  if (!amount.value) {
    result.value = '';
    return;
  }
  
  const numericAmount = parseFloat(amount.value);
  if (isNaN(numericAmount)) return;

  if (activeTab.value === 'Compra') {
    result.value = exchangeStore.convertToSoles(numericAmount).toFixed(2);
  } else {
    result.value = exchangeStore.convertToDollars(numericAmount).toFixed(2);
  }
});

El proyecto utiliza TypeScript para garantizar la seguridad de tipos:

// Tipos para las tasas de cambio
interface ExchangeRates {
  purchase_price: number;
  sale_price: number;
  last_updated: string;

// Función de conversión con tipos
type ConversionDirection = 'toPEN' | 'toUSD';

function convertAmount(
  amount: number, 
  rates: ExchangeRates, 
  direction: ConversionDirection
): number {
  if (direction === 'toPEN') {
    return amount * rates.purchase_price;
  } else {
    return amount / rates.sale_price;
  }
}

3. Preguntas de Código

3.1 Función handleInput

La función handleInput se encarga de validar y formatear la entrada del usuario:

const handleInput = (event: Event) => {
  const input = event.target as HTMLInputElement;
  let value = input.value;
  
  // Solo permite números y un punto decimal
  value = value.replace(/[^\d.]/g, '');
  
  // Maneja múltiples puntos decimales
  const parts = value.split('.');
  if (parts.length > 2) {
    value = `${parts[0]}.${parts.slice(1).join('')}`;
  }
  
  // Limita a 2 decimales
  if (parts.length === 2) {
    parts[1] = parts[1].substring(0, 2);
    value = parts.join('.');
  }
  
  // Actualiza el valor
  amount.value = value;
};

Esta función garantiza que la entrada del usuario sea siempre un número válido con hasta dos decimales.

3.2 Lógica de Conversión

// En store/exchange.ts
function convert(amount: number, from: 'USD' | 'PEN', to: 'USD' | 'PEN'): number {
  if (from === to) return amount;
  
  if (from === 'USD' && to === 'PEN') {
    return amount * purchasePrice.value;
  } else {
    return amount / salePrice.value;
  }
}

// Uso en el componente
const result = computed(() => {
  if (!amount.value) return '';
  
  const numericAmount = parseFloat(amount.value);
  if (isNaN(numericAmount)) return '';
  
  return exchangeStore.convert(
    numericAmount,
    activeTab.value === 'Compra' ? 'USD' : 'PEN',
    activeTab.value === 'Compra' ? 'PEN' : 'USD'
  ).toFixed(2);

3.3 Manejo de Suscripciones

La suscripción a las tasas de cambio se maneja de la siguiente manera:

// En store/exchange.ts
import { onUnmounted, ref } from 'vue';
import { doc, onSnapshot } from 'firebase/firestore';
import { db } from '@/firebase/config';

export const useExchangeStore = defineStore('exchange', () => {
  // Estado
  const purchasePrice = ref(0);
  const salePrice = ref(0);
  const loading = ref(true);
  const error = ref(null);
  let unsubscribe: (() => void) | null = null;

  // Inicializar la suscripción
  function subscribeToRates() {
    const ratesRef = doc(db, 'rates', 'current');
    
    unsubscribe = onSnapshot(
      ratesRef,
      (doc) => {
        if (doc.exists()) {
          const data = doc.data();
          if (data) {
            purchasePrice.value = data.purchase_price || 0;
            salePrice.value = data.sale_price || 0;
            loading.value = false;
            error.value = null;
          }
        } else {
          error.value = 'No se encontraron tasas de cambio';
          loading.value = false;
        }
      },
      (err) => {
        console.error('Error en la suscripción:', err);
        error.value = 'Error al cargar las tasas de cambio';
        loading.value = false;
      }
    );
  }

  // Limpiar la suscripción
  function unsubscribeFromRates() {
    if (unsubscribe) {
      unsubscribe();
      unsubscribe = null;
    }
  }

  // Limpiar al desmontar
  onUnmounted(() => {
    unsubscribeFromRates();
  });

  return {
    purchasePrice,
    salePrice,
    loading,
    error,
    subscribeToRates,
    unsubscribeFromRates
  };
});

Esta implementación asegura que:

  • Las suscripciones se limpian correctamente
  • Los errores se manejan apropiadamente
  • El estado de carga se actualiza correctamente
  • Los cambios en las tasas se reflejan en tiempo real

Watch - Para efectos secundarios y operaciones asíncronas:

4. Decisiones de Arquitectura

¿Por qué se eligió Vue 3 Composition API en lugar de Options API o Nuxt?

Vue 3 Composition API vs Options API:

Se eligió Composition API por varias razones clave:

  • Mejor organización del código: La lógica relacionada se agrupa por características en lugar de dividirse entre data, methods, computed, etc.
  • Reutilización de código: Permite extraer y reutilizar lógica entre componentes mediante composables.
  • Mejor tipado con TypeScript: La Composition API ofrece una mejor integración y soporte de tipos.
  • Más flexible: Facilita la extracción y composición de lógica compleja.

Ejemplo de ventaja con Composition API:

// Con Composition API
import { ref, computed } from 'vue';

export function useCurrencyConverter() {
  const amount = ref('');
  const rate = ref(3.8);
  
  const convertedAmount = computed(() => {
    return parseFloat(amount.value) * rate.value;
  });
  
  return { amount, rate, convertedAmount };
}

// Uso en cualquier componente
const { amount, rate, convertedAmount } = useCurrencyConverter();

¿Por qué no se usó Nuxt?

No se utilizó Nuxt porque:

  • Complejidad innecesaria: Para una aplicación SPA relativamente simple como un conversor de monedas, Nuxt añadiría complejidad sin beneficios significativos.
  • Rendimiento: No se requería SSR (Server-Side Rendering) ni SSG (Static Site Generation) para este caso de uso.
  • Mantenimiento: Un proyecto más pequeño es más fácil de mantener sin la capa adicional de abstracción que proporciona Nuxt.
  • Rendimiento inicial: La aplicación se carga rápidamente sin necesidad de las optimizaciones de Nuxt.

Cuándo sí usar Nuxt:

  • Aplicaciones que necesitan SEO mejorado (SSR/SSG)
  • Proyectos grandes con múltiples rutas y layouts
  • Cuando necesitas las convenciones de estructura de archivos de Nuxt
  • Para aprovechar módulos de Nuxt como Nuxt Image, Nuxt Content, etc.
